# Timber v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Execution mode for this session:** tasks 1–15 are independent (disjoint files) and are dispatched as parallel subagents; Task 16 (integration) runs after them. Build agents do NOT run `git commit` (parallel index-lock contention); the controller commits between phases. Subagents follow superpowers:test-driven-development per task.

**Goal:** Build Timber per [PRD.md](../../../PRD.md): a framework-free central log service — WAL-first ingest over pure `node:http`, background flusher to MongoDB, query/stats APIs, static UI, bench + kill-test acceptance.

**Architecture:** Single Node 22 process. Hot path: parse → validate → enrich → append to NDJSON WAL segment (group-commit fsync ≤50 ms) → `202`. Background flusher reads WAL from a checkpoint, `insertMany`s into Mongo (`ordered:false`, idempotent `_id`), advances checkpoint only after ack. Boot replay = flusher starting from the persisted checkpoint. Query API translates URL filters to Mongo queries; stats via aggregation (`$dateTrunc`, `$percentile`).

**Tech Stack:** Node 22 ESM, only runtime dep `mongodb` (v6 driver). Tests: `node:test` + `node:assert/strict`. UI: one static HTML file, vanilla JS. MongoDB server ≥ 7.0 required (`$percentile`).

## Decisions & deviations (approved interpretations of the PRD)

1. **`_id` derivation includes a per-process sequence.** `sha256(app \n receivedAt \n seq \n JSON(envelope))` (first 32 hex chars). PRD says hash of app+receivedAt+body; identical bodies in the same ms would collide and silently drop. The WAL stores the **final doc including `_id`**, so replay idempotency (the actual PRD goal) is fully preserved.
2. **`message` > 512 chars → silently truncated** to 512 (senders must not lose a log over a long line). `data` > 16 KB serialized → replaced with `{_truncated:true, _originalBytes:<n>, _head:<first 4096 chars of serialized JSON>}` per §5.4.
3. **Unknown top-level envelope keys → 400** (catches sender typos early; §6.1 wants senders to fix-then-retry).
4. **Batch > 500 events → 413**; body > 1 MB → 413; empty array → 400. Single object allowed (treated as batch of 1).
5. **Read-mode key on `POST /v1/logs` → 403** (`401` is reserved for unknown keys).
6. **Queries while Mongo is unreachable → 503** `{error:"storage unavailable"}`. Ingest stays up regardless (PRD §7.3).
7. **Dates:** in Mongo, `receivedAt`/`expiresAt` are BSON `Date` (needed for TTL, `$dateTrunc`, range scans). In WAL lines and JSON API responses they are ISO-8601 strings. The flusher revives ISO → `Date` at insert time. `ts` (sender clock) stays an untouched string.
8. **Transports (PRD T4) are out of scope for this repo** — §8 places them in the app repos. This repo ships T1, T2, T3, T5 and the buildable half of T6 (Dockerfile, compose, CI, USAGE.md).
9. **`node:cluster` flag:** `TIMBER_CLUSTER=<n>` forks n workers, each with WAL subdir `<walDir>/worker-<i>`. Off by default. Smoke-tested only.
10. **WAL writer never appends to a pre-existing segment.** On boot it opens a fresh segment at `maxExistingSeq+1` — sidesteps torn-tail append corruption.

### C5. WAL format — **the most load-bearing contract**

- Directory `walDir`, segment files `seg-<SEQ>.ndjson` where SEQ = 13-digit zero-padded positive int (`seg-0000000000001.ndjson`). Strictly increasing, no reuse.
- Each line: `JSON.stringify(doc) + '\n'` — doc per C3 `enrich` (ISO strings, includes `_id`).
- `checkpoint.json`: `{"segmentSeq": <int>, "offset": <int bytes consumed in that segment>, "updatedAt": "<ISO>"}`. Written atomically: write `checkpoint.json.tmp`, fsync file, rename over. Missing/corrupt file ⇒ `{segmentSeq: 0, offset: 0}`.
- Torn tail (no trailing `\n` at active segment end): reader must stop before it and report `atEnd`; never advance the checkpoint past incomplete bytes. A *complete* line that fails `JSON.parse` (corruption) is skipped and counted, checkpoint may advance past it.

```js
// wal/writer.js
export async function createWalWriter({dir, fsyncMs, segmentMaxBytes, budgetBytes, retainHours}) -> {
  append(docs) -> Promise<void>, // serialized internally (promise-chain queue); single fs write per call;
                                 // resolves after write() returns (OS buffer — NOT after fsync);
                                 // rotates first if active size ≥ segmentMaxBytes
  forceFsync() -> Promise<void>, // also: a dirty-flag timer fsyncs every fsyncMs
  totalBytes() -> number,        // sum of sizes of all seg-*.ndjson in dir (tracked, cheap)
  overBudget() -> boolean,       // totalBytes() >= budgetBytes
  activeSegmentSeq() -> number,
  janitor(checkpoint) -> Promise<{deleted: string[]}>,
    // delete segments with seq < checkpoint.segmentSeq AND mtime older than retainHours
  close() -> Promise<void>       // stop timer, final fsync, close fd
}
// On creation: mkdir -p; new segment at maxExistingSeq+1 (never append to old files).

// wal/checkpoint.js
export async function loadCheckpoint(dir) -> {segmentSeq, offset}
export async function saveCheckpoint(dir, {segmentSeq, offset}) -> Promise<void> // atomic per above

// wal/reader.js
export async function readWal(dir, checkpoint, maxDocs) -> {
  docs: object[],                       // parsed docs, ≤ maxDocs
  nextCheckpoint: {segmentSeq, offset}, // past consumed bytes (incl. skipped corrupt lines)
  atEnd: boolean                        // true ⇒ nothing more readable right now
}
// Walks segments with seq ≥ checkpoint.segmentSeq in order; checkpoint segment may have been
// janitor-deleted ⇒ start at the next existing seq with offset 0. Fills up to maxDocs, may span segments.
// When a segment is exhausted AND a later segment exists, nextCheckpoint moves to {laterSeq, 0}.
export async function backlogBytes(dir, checkpoint) -> number
// bytes after checkpoint: (size(cpSeg) - offset, floor 0) + sizes of all later segments
```

### C6. Flusher — `src/flusher.js`

```js
export function createFlusher({walDir, getCollection, batchSize, intervalMs, log}) -> {
  start(), stop() -> Promise<void>,
  status() -> {running, caughtUp, lastError: string|null, flushedTotal: number}
}
```
Loop: `loadCheckpoint` once at start, keep in memory → `readWal(walDir, cp, batchSize)` → if `docs.length`:
revive `receivedAt`/`expiresAt` to `Date` → `getCollection()` (null ⇒ wait `intervalMs`, retry; stays caughtUp=false) →
`insertMany(docs, {ordered:false})`; on error: if every `err.writeErrors[i].code === 11000` (also check `err.code === 11000` single form) treat as success (replay dupes); else record `lastError`, exponential backoff 1s→30s, do **not** advance checkpoint → on success `saveCheckpoint(nextCheckpoint)` + `flushedTotal += docs.length` → continue immediately. `atEnd && !docs.length` ⇒ `caughtUp = true`, sleep `intervalMs`. `stop()` resolves after the in-flight cycle finishes. Boot replay is simply `start()` from the persisted checkpoint.

### C8. Query: logs — `src/query/logs.js`, `src/query/cursor.js`

```js
// cursor.js
export function encodeCursor({receivedAt /* Date|ISO */, id}) -> string // base64url(JSON {r: iso, i: id})
export function decodeCursor(s) -> {receivedAt: Date, id: string} | null

// logs.js
export function parseLogsQuery(searchParams, limits) -> {ok:true, value:{filter, limit}} | {ok:false, error}
export async function runLogsQuery(collection, {filter, limit}) -> {items, nextCursor: string|null}
```
Param → filter mapping (unknown param name ⇒ 400; all values are `searchParams` strings):
- `app`, `env` → exact string match.
- `level` → csv → `{$in:[...]}`; any token ∉ LEVELS ⇒ 400.
- `event` → prefix: `{event: {$regex: '^' + escapeRegex(v)}}`.
- `from`/`to` → `receivedAt: {$gte: Date, $lt: Date}`; value = ISO or epoch-ms digits; unparseable ⇒ 400.
- `ids.<key>` → `{'ids.<key>': v}` exact.
- `data.<path>` → exact; if `/^-?\d+(\.\d+)?$/` match → `{$in:[Number(v), v]}`; `'true'/'false'` → `{$in:[boolean, v]}`; else string.
- `data.<path>__gte` / `__lte` → numeric required (else 400) → `{$gte/$lte: Number(v)}` merged per path.
- `q` → `{message: {$regex: q, $options:'i'}}`; length > 256 ⇒ 400; `new RegExp(q)` throws ⇒ 400.
- `limit` → int 1..500, default 100.
- `cursor` → decodeCursor; null ⇒ 400; else AND keyset predicate
  `{$or:[{receivedAt:{$lt:c.receivedAt}}, {receivedAt:c.receivedAt, _id:{$lt:c.id}}]}`.
Run: `find(filter).sort({receivedAt:-1,_id:-1}).limit(limit+1)`; if limit+1 returned, drop last, `nextCursor=encodeCursor(lastKept)`. Serialize Dates → ISO strings in items. Response body: `{items, nextCursor}`.

### C9. Query: stats — `src/query/stats.js`

```js
export function parseStatsQuery(searchParams) -> {ok:true, value:{group:'hour'|'day', from:Date, to:Date, app?, event?}} | {ok:false, error}
// group default 'hour' (else 400); from default to−24h; to default now; app exact; event prefix; unknown param ⇒ 400
export function buildStatsPipeline(v) -> pipeline   // exported for unit tests
export async function runStats(collection, v) -> {group, from, to, buckets:[...]}
```
Pipeline (exact shape):
```js
[
 {$match: {receivedAt:{$gte:from,$lt:to}, ...(app&&{app}), ...(event&&{event:{$regex:'^'+escapeRegex(event)}})}},
 {$group: {
   _id: {$dateTrunc: {date:'$receivedAt', unit: group}},
   total:{$sum:1},
   debug:{$sum:{$cond:[{$eq:['$level','debug']},1,0]}}, // info/warn/error same pattern
   latencyP:{$percentile:{input:{$convert:{input:{$ifNull:['$data.latencyMs','$data.durationMs']},to:'double',onError:null,onNull:null}}, p:[0.5,0.95,0.99], method:'approximate'}},
   statusTotal:{$sum:{$cond:[{$ne:[{$convert:{input:'$data.status',to:'double',onError:null,onNull:null}},null]},1,0]}},
   statusErrors:{$sum:{$cond:[{$gte:[{$convert:{input:'$data.status',to:'double',onError:null,onNull:null}},400]},1,0]}},
   costUsd:{$sum:{$convert:{input:'$data.costUsd',to:'double',onError:0,onNull:0}}},
   inputTokens:{$sum:{$convert:{input:'$data.inputTokens',to:'double',onError:0,onNull:0}}},
   outputTokens:{$sum:{$convert:{input:'$data.outputTokens',to:'double',onError:0,onNull:0}}},
 }},
 {$sort:{_id:1}}
]
```
Post-process each bucket: `{bucket: ISO(_id), total, counts:{debug,info,warn,error}, latency: latencyP[0]==null?null:{p50,p95,p99}, errorRate: statusTotal? statusErrors/statusTotal : null, costUsd: round 6 dp, inputTokens, outputTokens}`.

### C12. Fake collection — `test/helpers/fake-collection.js` (test-only; e2e without Docker)

```js
export function createFakeCollection() -> {
  insertMany(docs, opts), find(filter) -> {sort(s) -> this, limit(n) -> this, toArray()},
```

### C13. Bench & kill-test — `bench/`

- `loadgen.js`: `export async function runLoad({url, key, durationMs, concurrency, batchSize, eventFactory})` → `{sent, accepted, acked, errors, throughputPerSec, ackP50Ms, ackP99Ms}`. Plain `node:http` with `new http.Agent({keepAlive:true, maxSockets:concurrency})`; each worker loops POSTing batches, records per-request ack latency (hrtime).
- `ingest-bench.js` (CLI): spawns `node src/server.js` (temp WAL dir, `TIMBER_KEYS` with a write key, no Mongo needed — ingest path is WAL-only) unless `TIMBER_URL` env given; waits for `/healthz`; runs load (env-tunable: `BENCH_DURATION_MS` default 10000, `BENCH_CONCURRENCY` default 50, `BENCH_BATCH` default 20); prints report; exits 1 if `throughput < BENCH_MIN_RATE` (default 2000) or `ackP99Ms > BENCH_MAX_P99` (default 25). `BENCH_SMOKE=1` ⇒ duration 2000, min rate 500, p99 100 (CI-friendly).
- `kill-test.js` (CLI): temp WAL dir; spawn server; stream batches where every event carries `ids: {runId, n}` (n unique); track which batches got 202 (acked set); after ~1.5 s `child.kill('SIGKILL')` mid-stream; then:
  - **wal mode** (no `MONGODB_URI`): scan WAL segments directly (read NDJSON), assert every acked `n` present, assert no duplicate `n`.
  - **mongo mode** (`MONGODB_URI` set): restart server same WAL dir; poll `/healthz` until `flusher.caughtUp`; via `mongodb` driver count docs `{'ids.runId': runId}` — assert every acked `n` present and distinct-n count == doc count (no dupes). Then **idempotency proof**: stop server, reset `checkpoint.json` to `{segmentSeq:<first>, offset:0}`, restart, wait caughtUp, assert count unchanged (replay upserted, not duplicated).
  - exits 0/1 with a printed verdict line `KILL-TEST PASS|FAIL ...`.
